Contribution Guide

If you follow this tutorial but find yourself stuck, please submit an issue in the GitHub repository and include all steps you have taken and the full traceback to the error you've encountered so we may improve this page.

Getting Started

This is a basic tutorial on how to contribute to the website if you've never used GitHub, Git, or written any code for a website before. You can either use a remote environment via GitHub, or use your local machine.

Skip to "Setting up Workspace" Tutorial
Warning: Once you're done using the codespace, please make sure to either stop or delete it. This is done by clicking on Code, clicking on the three dots next to your codespace, and choosing the appropriate action. If GitHub is stuck on finding your codespace, refresh the page and try again.

Work on Your Local Machine

Install Anaconda

It is advised to use a context manager and an IDE (integrated development environment) to avoid many problems that arise when setting up your environment. We recommend using Anaconda, which installs the conda context manager.

winget install -e --id Git.Git
winget install -e --id Anaconda.Miniconda3
winget install -e --id Microsoft.VisualStudioCode

Set up Conda Environment

  1. Create a new Conda environment: 
    conda create -n CAPLEX python
  2. Activate your environment: 
    conda activate CAPLEX
  3. Install mkdocs-material: 
    pip install mkdocs-material

Setting up GitHub

  1. Set up a GitHub account.
  2. Fork the GitHub repository.
  3. Clone your fork in VSCode.
  4. Make changes and create a pull request.

Work in a Remote Environment

To create a new codespace (which contains the devcontainer), navigate to a GitHub branch containing a devcontainer.json file and start a new codespace.
